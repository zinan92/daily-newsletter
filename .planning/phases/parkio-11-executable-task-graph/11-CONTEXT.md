# Context: Phase 11 Executable Task Graph Foundation

The owner wants future building projects decomposed into atomic units, merged into execution threads by dependency, and claimable by idle agents. This is a task graph problem before it is an n8n problem.

The source of truth for this phase is repo-local JSON, because it is easy to validate, diff, commit, and test without adding runtime dependencies.

This phase does not sync GitHub Issues, does not run n8n, and does not change the Daily Inbox production pipeline.
